# Korus Web UI 精致化优化计划

## Goal

把 Korus Web Console 从“功能可用的后台界面”提升为“有明确产品气质、视觉统一、交互顺滑的企业级 Agent Control Plane”。本轮先做 UI/UX 规划，不直接改代码；确认后按 slice 实施、验证、提交并推送。

## Current Context

- Repo: `/home/windosx/dev/korus`
- Web stack: React + TypeScript + Vite + Tailwind CSS v4 + TanStack Query + i18n + React Flow。
- 最近提交：`7aa170b fix: align studio CRD sync and docs`，`main` 已推送。
- 当前 UI 已有基础设计语言：IBM Plex Sans / JetBrains Mono、teal accent、glass `surface`、深色 sidebar、浅色 grid background。
- 主要问题不是功能缺失，而是整体视觉层级、布局节奏、信息密度和 Studio 体验还偏“工程后台”，不够精致。

## Design Direction

建议采用 **industrial control plane / cybernetic workspace** 风格：

- 稳定、工程化、可信，而不是花哨 SaaS 模板。
- 视觉关键词：深色控制台骨架、浅色工作区、teal/emerald 能量线、细网格、低饱和灰阶、精密面板、状态灯、拓扑感。
- 保留 Korus 作为 K8s-native control plane 的气质：像“运行中枢”，而不是普通 CRUD 管理台。

## Success Criteria

1. 全局视觉系统统一：按钮、卡片、表格、header、sidebar、forms、badges 使用一致 token 和状态语言。
2. 首屏更有产品感：导航、页面标题、关键状态、空状态、loading 不再像默认后台模板。
3. Studio 明显升级：workflow canvas / pattern selector / model config / binding panels 更接近专业编排工作台。
4. 不破坏现有功能、路由、i18n、accessibility 和响应式。
5. 每个 slice 后验证：`npx tsc --noEmit`、`npm run build`、i18n key scan；最终还跑 `make ci`、`make helm-lint`、`make helm-template`、`git diff --check`。

## Risks / Tradeoffs

- **一次改太多风险高**：建议 slice-by-slice，每个 slice 可独立提交。
- **Tailwind class 膨胀**：应把重复组合沉淀到 `index.css` component layer 或 shared component variants。
- **视觉重构可能影响 accessibility**：尤其 focus、contrast、keyboard row navigation、modal focus trap，要逐项复验。
- **Studio canvas 最复杂**：避免同时重构状态模型；第一轮只做视觉和可见性，不改 graph 数据结构。
- **i18n 漏洞**：所有新增 UI 文案必须同时写 en/zh，并跑 literal `t()` scan。

## Recommended First Implementation Batch

建议第一批只做 **Slice 1 + Slice 2**：

- 先把 design token、shared components、AppShell/Sidebar/Header 打磨好。
- 这会立刻提升全局观感，同时风险低、影响面可控。
- 之后再进入 Studio 深水区。

第一批完成标准：
- 所有现有页面无需逐个改业务代码也能明显更统一。
- `npx tsc --noEmit` / `npm run build` / i18n scan 通过。
- 更新 `web/README.md` 的 UI conventions。
