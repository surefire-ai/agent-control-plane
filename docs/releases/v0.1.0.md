# v0.1.0 Release Notes

Korus v0.1.0 is the first Kubernetes-native MVP release. It
establishes the control-plane contract for declaring an Agent, compiling it into
a deterministic revision, invoking it through a gateway, running it through a
Kubernetes Job worker backend, and recording the result back into `AgentRun`
status.

## Highlights

- Adds Kubernetes API types and CRDs for `Agent`, `AgentRun`,
  `PromptTemplate`, `KnowledgeBase`, `ToolProvider`, `MCPServer`,
  `AgentPolicy`, and `AgentEvaluation`.
- Adds an Agent compiler that validates static cross-resource references,
  produces `Agent.status.compiledArtifact`, and records
  `Agent.status.compiledRevision`.
- Adds the AgentRun lifecycle with `Pending`, `Running`, `Succeeded`, and
  `Failed` phases.
- Adds an invoke gateway that accepts Agent invocation requests and creates
  `AgentRun` resources.
- Adds a Kubernetes Job runtime backend that launches a worker Job, passes the
  compiled artifact, parses structured worker output, and writes output plus
  trace references to `AgentRun.status`.
- Makes `runtime.engine=eino` and `runtime.runnerClass=adk` the default runtime
  direction.
- Adds an internal worker runner boundary with `EinoADKPlaceholderRunner` as the
  first implementation.
- Adds Dockerfiles for controller-manager and worker images.
- Adds CI checks, GHCR image publishing workflows, and a development Helm chart
  skeleton.
- Adds EHS hazard-identification sample resources for local end-to-end
  validation.

## Images

The release tag is expected to publish:

- `ghcr.io/surefire-ai/korus-controller-manager:v0.1.0`
- `ghcr.io/surefire-ai/korus-worker:v0.1.0`

The image publishing workflow also emits `sha-*` traceability tags.

## Install

Kustomize:

```bash
kubectl apply -k config/default
```

Helm development chart:

```bash
helm upgrade --install korus charts/korus \
  --namespace korus-system \
  --create-namespace \
  --set controllerManager.image.tag=v0.1.0 \
  --set controllerManager.worker.image.tag=v0.1.0
```

## Verify

Run local checks:

```bash
make test
make build
make helm-lint
make helm-template
```

Run the Kubernetes smoke test described in
`docs/releases/v0.1.0-readiness.md`.

## Known Limits

- Runtime execution is still a structured placeholder, not real Eino execution.
- Gateway authentication, authorization, rate limiting, and idempotency are not
  implemented.
- AgentRun cancellation, timeout, retry, and idempotency semantics are not
  implemented.
- Durable artifact and trace storage after worker Pod cleanup is not
  implemented.
- The Helm chart is a development and E2E install path; it is not yet packaged
  and published as the official installation artifact.
- CRD schema validation and admission checks are intentionally light.

## Next

Phase 2 focuses on replacing the placeholder worker with a real Eino-based
runtime while preserving the Kubernetes-native control-plane contract.
