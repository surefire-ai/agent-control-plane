# v0.1.0 Readiness Checklist

This checklist captures the Phase 1 exit path for the first public development
release of Korus.

Release notes live in `docs/releases/v0.1.0.md`.

## Release Scope

v0.1.0 is a Kubernetes-native MVP. It should prove that one Kubernetes-declared
agent can be compiled, published, invoked through the gateway, executed through
the Kubernetes Job runtime backend, and reported back with output, trace
reference, and exact agent revision identity.

The release does not include the real Eino runtime worker. The worker still uses
the `EinoADKPlaceholderRunner` behind the internal runner boundary.

## Required Checks

Run these before tagging:

```bash
make test
make build
make helm-lint
make helm-template
make docker-build
```

Expected result:

- All Go packages pass.
- `cmd/controller-manager` and `cmd/worker` build.
- The development Helm chart lints and renders with CRDs.
- The controller-manager and worker images build for `ghcr.io/surefire-ai`.

## Kubernetes Smoke Test

Use a local Kubernetes cluster such as OrbStack.

Install the control plane with Kustomize:

```bash
kubectl apply -k config/default
kubectl apply -f config/samples/ehs
kubectl -n korus-system rollout status deployment/korus-controller-manager --timeout=120s
```

Or install the development Helm chart:

```bash
helm upgrade --install korus charts/korus \
  --namespace korus-system \
  --create-namespace
kubectl apply -f config/samples/ehs
kubectl -n korus-system rollout status deployment/korus-controller-manager --timeout=120s
```

Validate the EHS sample:

```bash
kubectl -n ehs get agent ehs-hazard-identification-agent \
  -o jsonpath='{.status.phase}{"\n"}{.status.compiledRevision}{"\n"}{.status.endpoint.invoke}{"\n"}'
```

Expected result:

- The Agent reaches `Published`.
- `status.compiledRevision` is populated.
- `status.endpoint.invoke` points to the `windosx.com/v1alpha1` invoke path.

Invoke the Agent through the gateway and confirm the resulting `AgentRun`:

```bash
kubectl -n korus-system port-forward svc/korus-gateway 8082:8082
```

```bash
curl -sS -X POST \
  http://127.0.0.1:8082/apis/windosx.com/v1alpha1/namespaces/ehs/agents/ehs-hazard-identification-agent:invoke \
  -H 'Content-Type: application/json' \
  -d '{"input":{"task":"identify_hazard","payload":{"text":"巡检发现配电箱门打开，现场地面有积水。"}},"execution":{"mode":"sync","source":"v0.1.0-smoke"}}'
```

Expected result:

- The gateway returns `status: accepted`.
- A new `AgentRun` is created in namespace `ehs`.
- The `AgentRun` reaches `Succeeded`.
- `status.agentRevision` matches the Agent `status.compiledRevision`.
- `status.output.worker.status` is `succeeded`.
- `status.output.compiledArtifact.runtimeEngine` is `eino`.
- `status.output.compiledArtifact.runnerClass` is `adk`.
- `status.traceRef.podName` is populated.

## Image Publishing

The repository publishes images through `.github/workflows/publish-images.yml`.

Before tagging:

- Confirm `origin` points to `github.com:surefire-ai/korus`.
- Confirm `Makefile`, `config/default`, `config/manager`, and the chart values
  point to `ghcr.io/surefire-ai`.
- Confirm the `main` branch CI workflow passes.

Tagging:

```bash
git tag v0.1.0
git push origin v0.1.0
```

If the tag already exists, do not rewrite it unless the release is intentionally
being replaced.

Expected result:

- `ghcr.io/surefire-ai/korus-controller-manager:v0.1.0` is published.
- `ghcr.io/surefire-ai/korus-worker:v0.1.0` is published.
- Both images also receive `sha-*` traceability tags.

Current verification after the Korus image rename:

- Confirm `korus-controller-manager:v0.1.0` anonymous manifest inspection before
  tagging.
- Confirm `korus-worker:v0.1.0` anonymous manifest inspection before tagging.
- Confirm `korus-controller-manager:main` anonymous manifest inspection after
  the main branch publishing workflow completes.
- Confirm `korus-worker:main` anonymous manifest inspection after the main
  branch publishing workflow completes.

Optional local manifest check:

```bash
docker manifest inspect ghcr.io/surefire-ai/korus-controller-manager:v0.1.0
docker manifest inspect ghcr.io/surefire-ai/korus-worker:v0.1.0
```

If GHCR returns `unauthorized`, confirm package visibility and local registry
authentication before treating the image as missing.

## Known Limits

- Runtime execution supports real Eino ADK graph execution and iterative ReAct loops.
- Gateway authentication (Bearer token) and rate limiting (token bucket) are implemented.
- AgentRun lifecycle semantics — cancellation, timeout, and retry — are implemented.
- SubAgent composition via gateway HTTP calls is supported.
- Worker output is summarized in `AgentRun.status`; durable artifact and trace
  storage after Pod cleanup is not implemented.
- The Helm chart is a development and E2E install path. It is not yet packaged
  and published as the official installation artifact.
- CRD schema validation and admission checks are still intentionally light.

## Exit Decision

Phase 1 can be considered complete when all required checks pass, the
Kubernetes smoke test succeeds through either Kustomize or Helm, and the
`v0.1.0` tag publishes both images successfully.
